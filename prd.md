# Ports — 产品需求文档（PRD）

## 1. 项目概述

| 项目 | 说明 |
| --- | --- |
| 应用名称 | Ports |
| 平台 | macOS |
| 应用类型 | Menu Bar App（菜单栏常驻应用） |
| 核心功能 | 查看并管理系统当前占用的网络端口，支持快速搜索与一键关闭进程 |

Ports 是一款常驻 macOS 菜单栏的轻量工具，用于查看当前系统中正在监听/占用的网络端口列表，并支持对占用端口的进程进行搜索与关闭操作。

## 2. 核心需求清单

### 2.1 App 图标与菜单栏展示

- 应用启动后 **不在 Dock 中显示图标**（`LSUIElement = YES`），仅在菜单栏（Menu Bar）中显示。
- 菜单栏图标旁需要显示一个**数量标识（Badge）**。
  - **首版使用静态占位值 `8`**，后续接入真实端口数据后再动态更新。

### 2.2 弹框（Popover）展示

- 点击菜单栏图标后，弹出一个详情弹框。
- 弹框样式与 macOS 默认的 `NSPopover` 一致，**使用默认的弹框背景色**（系统半透明材质背景）。
- 弹框尺寸：**宽度 300，高度 500**（固定尺寸）。

### 2.3 弹框整体布局

弹框自上而下分为三个区域：**Header / Body / Footer**。

| 区域 | 高度（参考） | 说明 |
| --- | --- | --- |
| Header | ~50 | 标题与搜索 |
| Body | 400（默认） | 端口列表，可滚动 |
| Footer | ~50 | 帮助与全局操作 |

> 整体高度 500 = Header（50）+ Body（400）+ Footer（50），数值为参考布局，最终以视觉合理为准。

### 2.4 Header（头部）

- 采用 **左右布局**：
  - **左侧**：标题 `Ports`。
  - **右侧**：一个搜索图标（放大镜），以及位于最右侧的**刷新按钮**（用于手动刷新一次端口列表）。
- 点击搜索图标后，在**图标的左侧**展开一个搜索输入框，用于快速搜索/过滤端口列表。
- **过滤维度：按端口名称匹配**（仅匹配端口名称字段，其余字段不参与过滤）。
- 再次点击图标可收起搜索框（交互细节可后续细化）。

### 2.5 Body（端口列表）

- 默认高度 400。
- 当列表内容超出该高度时，**支持滚动**。
- 列表每一行展示以下信息：

| 字段 | 说明 |
| --- | --- |
| 端口名称 | 端口对应的名称/协议标识 |
| PID | 占用该端口的进程 ID |
| 执行名称 | 以什么名称（命令/可执行文件）执行产生的该端口 |
| 项目目录 | 对应端口执行的项目目录路径 |

- 每一行的**末尾（最右侧）** 放置一个**关闭图标**，点击后可关闭该端口对应的进程。
  - **单行关闭无需二次确认**，点击即执行。
  - **关闭进程默认采用 `sudo` 执行**（即以管理员权限终止目标 PID），以应对权限不足的进程。

### 2.6 Footer（底部）

- 采用 **左右布局**：
  - **左侧**：问号图标 + 文案 `Need Help?`。点击后**打开系统默认浏览器**访问 `http://google.com`。
  - **右侧**：两个操作按钮：
    - `close all`：关闭全部端口对应的进程。**点击前需弹出二次确认弹窗**，确认后再执行。关闭同样采用 `sudo` 执行。
    - `Quit`：退出应用。

## 3. 交互流程

1. 应用启动 → 隐藏 Dock 图标 → 在菜单栏显示图标 + 数量 Badge（8）。
2. 用户点击菜单栏图标 → 弹出 300×500 的 Popover。
3. Popover 中：
   - 顶部可展开/收起搜索框，过滤端口列表。
   - 中部展示端口列表，支持滚动，每行可单独关闭。
   - 底部可访问帮助链接或执行全局关闭/退出。

## 4. 技术要点（建议）

- 技术栈以 **SwiftUI 为主**：使用 `NSStatusItem` + `NSPopover` 承载 SwiftUI 视图（通过 `NSHostingView`/`NSHostingController`）。
- `Info.plist` 中设置 `LSUIElement = YES` 以隐藏 Dock 图标。
- 端口与进程数据获取：可通过系统命令（如 `lsof -i -P -n`）或 `Darwin` API 获取监听端口及对应 PID、可执行文件、工作目录等信息。
- **数据范围**：包含 TCP 监听端口、UDP 端口、以及已建立连接（ESTABLISHED）的端口。
- **数据刷新机制**：后台自动每 30 秒刷新一次；同时在 Header 最右侧提供手动刷新按钮，可随时刷新一次。
- 关闭进程：通过 `sudo` 执行（如 `sudo kill <PID>`），以管理员权限终止目标进程。
- 打开默认浏览器：使用 `NSWorkspace.shared.open(_:)`。

## 5. 验收标准（Acceptance Criteria）

- [ ] 应用启动后 Dock 中不显示图标，仅菜单栏可见。
- [ ] 菜单栏图标旁正确显示数量 Badge（首版为 8）。
- [ ] 点击菜单栏图标弹出 300×500 的 Popover，使用默认背景色。
- [ ] Header 左右布局正确，搜索图标点击后可在其左侧展开搜索框；最右侧有刷新按钮可手动刷新一次。
- [ ] Body 列表展示端口名称、PID、执行名称、项目目录，内容超出可滚动。
- [ ] 列表数据后台每 30 秒自动刷新一次，且可手动刷新。
- [ ] 端口数据覆盖 TCP / UDP / 已建立连接。
- [ ] 每行末尾有关闭图标，点击以 `sudo` 关闭对应进程。
- [ ] Footer 左侧 `Need Help?` 点击可打开默认浏览器访问 google.com。
- [ ] Footer 右侧包含 `close all`（带二次确认）与 `Quit`，功能正常。

## 6. 已确认事项

1. **数量 Badge**：首版使用静态占位值 `8`，后续再接入真实端口数据动态更新。
2. **关闭确认**：单行关闭无需二次确认；`close all` 需弹出二次确认弹窗。
3. **关闭方式**：关闭进程默认采用 `sudo` 执行（管理员权限）。
4. **技术栈**：以 SwiftUI 为主，菜单栏承载使用 `NSStatusItem` + `NSPopover` + `NSHostingView`。
5. **搜索过滤**：仅按端口名称字段匹配。
6. **数据刷新**：后台每 30 秒自动刷新 + Header 最右侧手动刷新按钮。
7. **数据范围**：包含 TCP 监听、UDP、已建立连接（ESTABLISHED）。
